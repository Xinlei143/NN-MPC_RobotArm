# 项目发展思路（历史研究叙事）

> **历史实现说明**：本文保留 residual MPC 之前的 reference-increment / `delta_q_ref` 研究表述，用于记录论文定位的演进。当前默认控制器不再直接优化无锚定 `delta_q_ref`：它以 projected IK nominal `q_nom` 为基线，优化有界 residual `r`，并以 `lowest_cost` 比较 direct nominal、best 和 mean。当前实现规范见 [README](../README.md)、[CostFunction](CostFunction.md) 和 [MPC 伪代码](mpc-pseudocode.md)。本文的研究问题、black-box actuator-reference dynamics 和 OOD 讨论仍可作为未来工作依据。

可以把整篇论文的逻辑压缩成一句话：

**本文解决的是“位置控制工业机械臂中，期望轨迹 `q_des`、执行器参考命令 `q_ref` 与实际关节运动 `q` 不一致时，如何学习闭环 actuator-reference dynamics，并将其用于采样式 MPC 控制，同时缓解 MPC 闭环运行带来的数据分布偏移问题”。**

这个问题比“用神经网络做机械臂 MPC”更具体，也更容易和已有工作区分。你当前项目已经具备这个框架雏形：每个控制步从 MuJoCo 获取当前状态，生成未来参考轨迹，让 CEM 采样未来 `delta_q_ref`，用 learned dynamics 预测候选未来状态，用 cost 评分，只执行最优序列第一步，再回到真实环境闭环滚动。

## 1. 整篇论文要解决的新问题是什么？

论文不要说自己解决“机械臂轨迹跟踪问题”，因为这个太宽、太旧。更好的问题定义是：

**在只开放 position-control interface 的工业机械臂中，控制器不能直接施加理想力矩，也不能假设发送 `q_des` 后机械臂就会精确跟踪。真实控制输入其实是 actuator reference `q_ref`，而系统输出是实际状态 `q, dq`。因此需要学习的是从历史状态和未来 actuator reference sequence 到未来状态的闭环映射，而不是传统意义上的刚体动力学方程。**

你的项目里本来就明确区分了 `q_des` 和 `q_ref_command`：`q_des` 是希望机械臂跟踪的参考轨迹，`q_ref_command` 才是真正发给 MuJoCo position actuator 的输入；当前控制不是直接发送 `q_des`，而是让 CEM 找一个更合适的 `q_ref` 来让实际关节状态尽量跟上 `q_des`。

这就是新问题的核心：

**不是“已知动力学下怎么做 MPC”，也不是“用 NN 代替动力学模型”，而是“在 position actuator 作为黑箱闭环低层控制器时，如何学习 actuator command 到实际关节运动的闭环动态，并用它做 MPC”。**

这个问题在工业机械臂、仿真机械臂、只开放位置接口的机器人平台中都合理。很多机器人控制论文默认 torque control 或完整动力学模型，而你的论文可以强调：实际很多平台只能发送关节位置参考，低层控制器、执行器延迟、阻尼、饱和和速度限制共同形成了一个复杂的闭环系统。

## 2. 论文的核心方法是什么？

你的方法可以叫：

**Actuator-Reference-Aware Learned CEM-MPC**

或者更完整一点：

**Closed-Loop Dynamics Learning and CEM-MPC for Position-Controlled Manipulators**

方法由四层组成。

第一层是 **actuator-reference-aware dynamics model**。模型输入不是简单的当前状态和力矩，而是历史状态、历史 actuator reference，以及候选未来 actuator reference sequence。输出是未来关节状态预测。项目结构中已经说明当前 position-control 数据里 `actions` 和 `q_ref` 都表示 actuator target joint angles，并且 learned dynamics 支持 MLP、GRU、Transformer、single-step target、rollout loss 和 learned rollout utilities。

第二层是 **CEM over reference increments**。你的 CEM 不直接优化绝对关节角，而是优化一串 `candidate_delta_q_ref`，再经过累积、限幅、rate limit、joint limit margin 等处理，转换成可执行的 `q_ref_sequences`，随后送入 learned dynamics 做 rollout。这个设计比“直接优化 q”更适合 receding-horizon 控制，因为它天然约束了动作变化率，也更容易保持控制平滑。

第三层是 **constraint-aware tracking cost**。你的代价函数不是只有 tracking error，而是包含关节位置误差、速度误差、actuator reference 与当前状态偏差、q_ref rate penalty、delta_q_ref rate penalty、terminal tracking error 和 joint-limit penalty。 这可以支撑你把论文写成控制论文，而不是单纯神经网络预测论文。

第四层是 **MPC-induced data aggregation and OOD analysis**。这是你最应该强调的地方。你的项目已有 `collect_mpc_data.py`、`evaluate_model_abc.py`、`analyze_ood_mpc.py`：分别用于把 MPC rollout 转换成训练数据、比较 A/B/C checkpoints、分析 MPC 访问到的 state/action 与训练分布之间的偏移。 你的伪代码也已经说明，`collect_mpc_data.py` 会复用 closed-loop MPC rollout，把实际 `state_t`、`actuator_q_ref_t`、`next_state_t` 等重新整理成 dynamics training dataset。

所以整篇论文的方法结构应该是：

**先用普通轨迹数据训练 Model A，再用 MPC-like 数据训练 Model B，然后用 Model A/B 运行闭环 MPC，采集 MPC 实际访问到的数据，训练 Model C，最后比较三类模型在闭环 MPC 中的 tracking error、failure rate、planning time、OOD fraction 和 robustness。**

这条逻辑比“我训练了一个 Transformer”强很多。

## 3. 论文最后应该呈现什么效果？

这里要严谨：在你真正跑完实验前，不能写具体提升百分比。但是如果你完成了前面说的全部内容，论文最后应该呈现四类结果。

第一，**相对直接发送 `q_des`，你的方法能显著降低高速或复杂轨迹下的 tracking error**。关键不是说“神经网络更准”，而是证明 CEM 会选择一个不同于 `q_des` 的 actuator reference `q_ref`，从而补偿 position actuator 的闭环响应滞后、平滑限制或动态误差。最有说服力的图应该同时画 `q_des`、direct `q_ref=q_des`、CEM-selected `q_ref` 和 actual `q`。

第二，**相对普通 offline learned dynamics，MPC-induced data aggregation 能改善闭环控制效果**。你应该证明 Model C 在闭环 tracking RMSE、max error、failure rate 上优于 Model A/B；即使 Model C 的 one-step validation loss 不一定最低，也要证明它在 MPC-induced distribution 上的 rollout error 更低。你的 `evaluate_model_abc.py` 本来就是为同一套 MPC 设置下比较多组 dynamics checkpoint 设计的。

第三，**OOD fraction 与闭环性能之间存在对应关系**。也就是说，当 MPC 访问的 state/action 偏离训练数据越远时，prediction-real gap 和 tracking error 越容易变大；经过 MPC-induced data aggregation 后，OOD fraction 应下降，closed-loop performance 应更稳定。你的 `analyze_ood_mpc.py` 已经定义了 state/action z-score、OOD fraction、failure rate 和 predicted-real gap 等指标。

第四，**扰动和约束实验中，方法比直接参考跟踪、PID/PD、普通 learned MPC 更稳**。这里包括 payload perturbation、actuator gain mismatch、reference frequency sweep、joint-limit stress test、initial-state perturbation。最终你要呈现的不是“永远最优”，而是“在 position-control closed-loop dynamics 与训练分布偏移存在时，本文方法能更稳定地维持跟踪性能，并显式减少动作抖动和限位违规”。

如果再完成 learning-guided CEM warm start，还可以加第五类结果：**在较少 CEM samples 或较短 planning time 下，learned proposal + CEM refinement 达到接近或优于普通 CEM 的 tracking performance**。这会让论文更接近 2025–2026 年 learning-guided MPC 的趋势。

## 4. 论文完整故事线可以这样写

### Introduction 的逻辑

第一段：机械臂轨迹跟踪通常依赖模型控制、MPC 或学习控制，但许多工业机械臂和仿真平台并不直接开放 torque input，而是通过 position reference command 控制。此时发送给执行器的 `q_ref` 与期望轨迹 `q_des` 并不等价。

第二段：传统 MPC 多假设可用解析动力学或 torque-level model；普通 NN-MPC 则常把神经网络作为系统 dynamics 预测器。但在 position-control interface 下，真正需要建模的是低层执行器闭环、参考命令、关节状态之间的复合动态。

第三段：更严重的是，learned dynamics 在 offline 数据上训练后，被 MPC 放进闭环优化时，会访问训练数据之外的 state/action distribution。这个 distribution shift 会导致多步预测误差放大，进而影响 MPC 控制性能。

第四段：本文提出 actuator-reference-aware learned CEM-MPC：学习闭环 actuator-reference dynamics，用 CEM 优化未来 `delta_q_ref`，通过 learned rollout 预测候选轨迹，并用 MPC-induced data aggregation 缓解闭环分布偏移。

### Method 的逻辑

方法部分分五小节最清楚：

1. Problem formulation：定义 `q_des`、`q_ref`、`q`、`dq`，说明 position-control setting。
2. Closed-loop dynamics learning：说明模型输入输出、history window、MLP/GRU/Transformer。
3. CEM-MPC over actuator reference increments：说明优化变量是 `delta_q_ref`，不是 torque，也不是直接 `q_des`。
4. Cost and constraints：tracking、smoothness、terminal、joint-limit、rate-limit。
5. MPC-induced data aggregation：Model A/B/C，OOD metrics，retraining pipeline。

### Experiments 的逻辑

实验部分不要按“模型一、模型二、模型三”排列，而要按审稿人关心的问题排列：

1. **Does optimizing actuator reference improve tracking?**
   比较 direct `q_ref=q_des`、filtered reference、PID/PD、learned CEM-MPC。

2. **Which learned dynamics works best in closed-loop MPC?**
   比较 MLP、GRU、Transformer；同时报告 one-step loss、多步 rollout error、closed-loop tracking error。

3. **Does MPC-induced data aggregation reduce distribution shift?**
   比较 Model A/B/C 的 OOD fraction、prediction-real gap、tracking RMSE。

4. **Is the controller robust under mismatch and constraints?**
   做 payload、actuator gain、reference frequency、joint-limit stress test。

5. **Can learning-guided CEM improve planning efficiency?**
   如果完成 proposal network，比较 no warm-start、shifted warm-start、learned proposal warm-start 在不同 sample budget 下的效果。

## 5. 和当前学术界相关工作的差异化

下面是最关键的 comparison framing。

### 5.1 相对传统 model-based MPC

传统 model-based MPC 的强项是约束处理清楚、理论结构成熟，尤其适合有准确动力学模型和 torque-level control 的机器人。2025 年仍然有 full-dynamics real-time NMPC 用于重型液压机械臂轨迹跟踪的工作，这类方法强调完整非线性动力学、执行器约束、实时求解和实验验证。([arXiv][1])

你的差异是：**你不假设可以直接使用准确解析动力学，也不假设能直接控制 torque，而是面向 position-control interface 学习 closed-loop reference-to-state dynamics。** 这使你的方法更适合那些只暴露关节位置目标接口、低层控制器不可完全建模的平台。

弱点也要诚实承认：你的理论约束保证不如 full-dynamics NMPC 强，实机说服力也不如真实液压机械臂实验。但你的优势是更贴近“黑箱低层位置执行器 + 数据驱动建模”的场景。

### 5.2 相对普通 NN-MPC / deep learning MPC

2025 年已经有 NN-MPC + active disturbance rejection 用于 4-DoF 机械臂轨迹跟踪的工作，其中神经网络用于 derivative-free dynamics prediction，MPC 在约束下做轨迹跟踪。([ScienceDirect][2]) 也有 attention-based GRU 与 event-driven NMPC 结合的工作，用 attention-GRU 建模复杂 inverse dynamics，再配合 NMPC 和增强状态估计。([ScienceDirect][3])

你的差异不是“我也用了神经网络”，而是三点：

第一，你学习的是 position actuator reference 到实际状态的闭环 dynamics，不是一般意义上的系统 dynamics 或 inverse dynamics。

第二，你不是只报告神经网络预测误差，而是把 one-step loss、多步 rollout error、closed-loop MPC tracking error、OOD fraction 连接起来，解释为什么某个模型在 MPC 中有效或失效。

第三，你引入 MPC-induced data aggregation，针对 MPC 实际访问的分布重新训练模型。这比普通 offline NN-MPC 更有研究问题意识。

所以你在 related work 里可以这样写：已有 NN-MPC 工作通常关注 neural dynamics 或 inverse dynamics 的预测精度及其在 MPC 中的应用，而本文进一步研究 learned dynamics 被 sampling-based MPC 闭环调用后产生的 distribution shift，并通过 MPC-induced data aggregation 改善闭环控制性能。

### 5.3 相对 Koopman-MPC / structured latent dynamics

Koopman-MPC 的核心思想是把非线性系统提升到高维 latent/lifted space 中，用近似线性模型做预测控制。2024–2025 年已有 input-augmented Koopman MPC、adaptive Koopman MPC、Meta-Koopman 等工作，重点是结构化、可优化、可泛化的动力学表示。([ScienceDirect][4])

你的差异是：**你不追求把系统变成线性 lifted dynamics，而是采用 sequence model/black-box learned dynamics 支持采样式 CEM rollout。** 这牺牲了一部分优化结构，但换来对 position actuator closed-loop behavior 的直接建模能力。你还可以把 Koopman 作为 baseline 或 related work，说明它更适合 structured predictive model，而你的重点是 actuator-reference-aware closed-loop rollout 和 data aggregation。

如果你加一个 Koopman baseline，会进一步增强说服力：即使 Koopman 是强相关方向，你仍可以证明在你的 position-control setting 下，GRU/Transformer + MPC-induced data 可能更适合复杂闭环参考响应。

### 5.4 相对 data-driven robust MPC / redundant manipulator MPC

ROBIO 2024 的 Data-Driven Robust MPC for Pose Tracking of Redundant Manipulators 明确提出用于冗余机械臂位姿跟踪的 data-driven robust MPC，考虑关节角、速度、加速度约束，并处理模型不确定性。([IEEE Xplore][5]) 你之前的论文分析也指出，这类工作强项是 redundant manipulator、end-effector pose tracking、多层级约束和 robust MPC，而不是神经网络闭环 actuator dynamics。

你的差异是：

**它关注“冗余机械臂如何在任务空间位姿跟踪中满足约束和鲁棒性”；你关注“位置执行器接口下，如何学习 actuator reference 到实际关节状态的闭环映射，并在采样式 MPC 中处理由闭环优化引起的数据分布偏移”。**

也就是说，你不应该和它硬拼 robust MPC 理论，而是把它放在 related work 的 constrained/robust predictive control 类别下。你的论文则属于 learned closed-loop dynamics + sampling-based MPC + data aggregation。

### 5.5 相对 null-space MPC / redundancy resolution

ROBIO 2024 的 Online Predictive Motion Control of Redundant Robots via MPC and Full Null Space Parameterization 使用 full null space parametrization 来确定 null-space basis 的最优线性组合，再用 MPC 生成冗余机器人的控制序列。([IEEE Xplore][6])

你的差异非常明确：**它解决冗余自由度和 secondary objective 的问题；你的项目不主打 redundancy resolution，而是主打 position actuator dynamics、CEM rollout 和闭环数据分布。**

这意味着你没必要临时加入复杂 null-space 控制。相反，你应该在论文里说明本文实验平台虽是工业机械臂 joint-space/reference-space tracking，但方法关注的是低层位置执行器闭环建模，可与 task-space planner 或 redundancy resolver 结合。

### 5.6 相对 learning-guided MPC / diffusion prior / proposal distribution

2025–2026 年一个明显趋势是：learning 不只是学习 dynamics，还用于 MPC 的 warm start、motion prior、risk model 或 sampling proposal。ROBIO 2025/公开扩展稿中的 learning-guided force-feedback MPC for robotic deburring 使用 diffusion-based motion priors 给 MPC 提供 robust initialization and adaptation，而 MPC 负责 force tracking、torque feasibility 和 collision avoidance。([arXiv][7]) 你之前的分析也指出，2025 年后更强的 learning+MPC 工作开始把 learning 用在 dynamics、residual、cost、reference、proposal distribution、warm start、risk model 等不同位置。

如果你完成 learning-guided CEM warm start，你的差异化会更强：

**已有 diffusion-prior MPC 面向 contact-rich deburring，强调力反馈、碰撞约束和接触任务；你的 learned proposal CEM 面向 free-space/joint-space position-controlled manipulator，强调 sample efficiency 和 actuator-reference tracking。**

你不需要声称比 diffusion prior 更强，只要证明在你的 setting 下，一个轻量 proposal network 能减少 CEM sample budget 或提升同等 budget 下的 tracking performance，就能把论文从普通 NN-MPC 提升到 learning-guided sampling-based MPC。

### 5.7 相对 model-based RL / PETS / CEM planning

CEM + learned dynamics 在 model-based RL 里不是新东西。PETS 这类早期工作已经用 probabilistic ensembles、trajectory sampling 和 MPC 进行 model-based control，并强调 sample efficiency。你必须避免让审稿人觉得你只是把 PETS/CEM-MPC 放到机械臂上。

你的区别应写成：

**本文不是泛化的 model-based RL benchmark，而是针对 position-controlled industrial manipulator 的 actuator reference interface，研究 `q_des`、`q_ref`、actual `q` 之间的闭环动态建模；同时将 MPC-induced OOD analysis 和 data aggregation 纳入控制实验。**

这才是你从“套用 CEM-MPC”变成“有具体机器人控制问题”的关键。

## 6. 论文贡献点建议写法

最终论文贡献建议写成三条，不要写五六条。

**Contribution 1:**
提出一种 actuator-reference-aware closed-loop dynamics learning formulation，用于 position-controlled manipulators。该模型学习历史状态与 actuator reference sequence 到未来关节状态的映射，显式区分 desired trajectory `q_des` 和 executable actuator reference `q_ref`。

**Contribution 2:**
提出 learned CEM-MPC over actuator reference increments。控制器在 receding horizon 内优化 `delta_q_ref` sequence，经限幅和平滑后转换为 actuator `q_ref`，并通过 learned rollout 与 constraint-aware cost 选择当前控制输入。

**Contribution 3:**
提出并验证 MPC-induced data aggregation 与 OOD diagnostics。系统比较 offline data、MPC-like data、MPC-induced data 对 learned dynamics 和 closed-loop MPC performance 的影响，并通过 OOD fraction、prediction-real gap、tracking error 等指标说明闭环分布偏移如何影响控制性能。

如果完成 learned proposal CEM，可以把第三条后半部分扩展为：

**Additionally, a learned CEM initialization module improves sampling efficiency under limited online planning budgets.**

但这最好作为增强贡献，不要抢主线。

## 7. 最终论文摘要可以这样组织

你可以按下面这个逻辑写摘要：

“Position-controlled robot manipulators expose actuator reference commands rather than direct torque inputs, making the closed-loop transition from desired joint trajectories to actual motion difficult to model accurately. Directly sending desired references often leads to tracking lag, overshoot, or constraint violations under aggressive trajectories and actuator mismatch. This paper proposes an actuator-reference-aware learned CEM-MPC framework for position-controlled industrial manipulators. A history-conditioned neural dynamics model predicts future joint states under candidate actuator reference sequences, while a CEM optimizer searches over reference increments and selects smooth, constraint-aware commands in a receding-horizon manner. We further analyze the closed-loop distribution shift induced by MPC and introduce MPC-induced data aggregation to improve dynamics learning under the state-action distribution actually visited by the controller. Experiments on an ABB IRB2400 MuJoCo platform compare direct reference tracking, filtered references, PID/PD baselines, MLP/GRU/Transformer dynamics, and A/B/C data aggregation variants under nominal, high-frequency, payload, actuator-mismatch, and joint-limit stress conditions. Results show that the proposed method improves tracking accuracy and robustness while reducing OOD rollout queries and maintaining smooth actuator references.”

注意最后一句目前还是模板，等你有数据后要改成真实数值，例如 “reduces RMSE by xx%”。

## 8. 整篇论文的“审稿人视角”判断

如果我是审稿人，我会这样理解你的论文：

**这不是一篇提出全新 MPC 理论的论文，也不是一篇纯神经网络预测论文。它的价值在于把一个常被忽略但实际存在的问题定义清楚：position-controlled manipulator 中，MPC 真正能优化的是 actuator reference，而不是理想力矩或理想轨迹；learned dynamics 被放入闭环 MPC 后会发生 distribution shift，因此需要 MPC-induced data aggregation 和 OOD diagnostics。**

这个定位是合理的，也比“NN-MPC for robot arm tracking”更容易被接受。

但是审稿人会重点看四个证据：

1. 你是否真的证明 direct `q_ref=q_des` 不够好，而你的方法能学出更合适的 `q_ref`。
2. 你是否真的证明 closed-loop OOD 会影响 MPC，而 data aggregation 能缓解。
3. 你是否有足够 baseline，不只是自己和自己比。
4. 你是否在扰动、约束、高频轨迹下仍然有效，而不是只在单一 sine trajectory 上有效。

只要这四点证据完整，差异化是成立的。没有这四点，论文会退化成普通 NN-MPC 仿真项目。
